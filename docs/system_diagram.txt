REPAIR LOOP
===========

  ITER 0 (cold start, runs once)
  ──────────────────────────────
  ┌─────────────────────────────────────────┐
  │  For each tag, detect outlier-heavy:    │
  │   e5_diversity > 0.4 OR                 │
  │   medoid-vs-trimmed-mean disagreement   │
  │   > 0.15                                │
  │                                         │
  │  Bootstrap centroid:                    │
  │   • outlier-heavy → medoid + core       │
  │     (rows with cos > 0.85 to medoid)    │
  │   • otherwise → trimmed mean (drop      │
  │     top/bottom 5% by sim-to-rough)      │
  │                                         │
  │  FREEZE iter-0 centroids and pairwise   │
  │  cosines. Used as hysteresis predicate  │
  │  for merge phase + absorber rescue.     │
  └─────────────────────────────────────────┘

                       │
                       ▼

  ITER k = 1..max_iter (default 8)
  ────────────────────────────────

  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  ┌─────────────────────────────────────────────────────┐   │
  │  │  PHASE A — REASSIGN                                 │   │
  │  │                                                     │   │
  │  │  For each kept row r in tag A:                      │   │
  │  │   1. Compute margin = cos(r, C[A]) − cos(r,         │   │
  │  │      best-other-centroid)                           │   │
  │  │   2. Cheap gates: margin < reassign_thresh AND      │   │
  │  │      gap > delta_move (+delta_hyst if back-move)    │   │
  │  │      AND moves_remaining > 0 AND best_other not     │   │
  │  │      untrusted                                      │   │
  │  │   3. Expensive gate (lazy): FAISS top-10 kNN        │   │
  │  │      tag-share. Need knn_share[best_other] >        │   │
  │  │      knn_share[current]                             │   │
  │  │   4. Sort candidates by gap, apply per-tag intake   │   │
  │  │      cap (10%); strongest moves win                 │   │
  │  │   5. Recompute affected centroids with annealed     │   │
  │  │      damping (50/50 → 70/30 → 85/15 → 100/0)        │   │
  │  └─────────────────────┬───────────────────────────────┘   │
  │                        │                                    │
  │                        ▼                                    │
  │  ┌─────────────────────────────────────────────────────┐   │
  │  │  PHASE C — MERGE TAGS  (only iter ≤ 2)              │   │
  │  │                                                     │   │
  │  │  For each tag pair (a, b):                          │   │
  │  │   1. cos(C[a], C[b]) > merge_cosine_thresh (0.90)   │   │
  │  │   2. Hysteresis: iter-0 frozen pairwise also > 0.90 │   │
  │  │   3. Mutual directed confusion > 0.50 in BOTH dirs  │   │
  │  │   4. kNN overlap (sampled 50+50 rows) > 0.50        │   │
  │  │                                                     │   │
  │  │  → MERGE: pick canonical via geometric composite    │   │
  │  │    score (row count + neighborhood support −        │   │
  │  │    medoid disagreement). Loser dissolves; rows      │   │
  │  │    reassign to canonical.                           │   │
  │  │                                                     │   │
  │  │  C runs BEFORE B so cross-tag pairs whose tags      │   │
  │  │  merge disappear from B's pair-set.                 │   │

PER-ROW REASSIGNMENT DECISION (PHASE A GATES, IN ORDER)
=======================================================

   Row r in tag A
        │
        ▼
   margin[r] = cos(r, C[A]) − cos(r, C[best_other])
        │
        ▼
   margin[r] < reassign_thresh (-0.03)? ─── NO ───► STAY
        │ YES
        ▼
   gap > delta_move (0.05)?              ─── NO ───► STAY
   (+0.02 if best_other == prior_tag)
        │ YES
        ▼
   moves_remaining[r] > 0?                ─── NO ───► FROZEN
        │ YES
        ▼
   best_other ∉ untrusted_tags?           ─── NO ───► STAY
        │ YES
        ▼
   knn_share[best_other] >                ─── NO ───► STAY
   knn_share[current]?
        │ YES
        ▼
   per-tag intake cap available?          ─── NO ───► DEFER
        │ YES (rank by gap, top-N apply)
        ▼
   ── REASSIGN ──
   A(r) ← best_other
   prior_tag[r] ← A
   moves_remaining[r] −= 1


CROSS-TAG NEAR-DUP TRIAGE (PHASE B)
===================================

   For each pair (r_a in tag_A, r_b in tag_B) with cos(r_a, r_b) > 0.97:
   (Phase C ran first, so pairs whose tags merged are already same-tag
   and not detected here.)

   |own_sim(r_a) − own_sim(r_b)| > 0.05?
        │
        ├── YES → WINNER-TAKE: drop the row with lower own_sim
        │
        └── NO  → DROP BOTH (last resort)

KEY DESIGN DECISIONS
====================

  • Pure E5 geometry only. No tag_answer.json, no tag-name patterns,
    no LLM. The user accepted: this optimizes for E5-routability, not
    truth-preserving taxonomy. Two tags with materially different
    answers will collapse if E5 says they're indistinguishable.

  • Phase order C → B (merge before drop) prevents stuck-deferred
    pairs that would never resolve under the original A → B → C order.

  • Iter-0 frozen centroids serve double duty: hysteresis predicate
    for merges, and absorber rescue (replace runaway-magnet centroid).

  • Annealed damping (heavy early, light late) prevents centroid
    swing during the high-churn early iters.

  • Per-row move budget = 3 + asymmetric δ_move/δ_hyst prevents
    ping-pong oscillation.

  • Lazy kNN-share: FAISS top-K computed only for cheap-pass candidates,
    not all rows. Scales linearly with candidate count, not corpus size.

  • Resource cap locked at module load: OMP/MKL/OpenBLAS/FAISS = 4
    threads, peak working set < 2 GB.
